E) SERVER/API NOTES (FOR REAL BUILD, NOT PREVIEW)
-------------------------------------------------
In production, the following MUST be server-authoritative:
- Status transitions (Scheduled/Live/Ended) with audit log
- Inventory enforcement and concurrency (atomic decrement)
- Pricing enforcement
- Idempotent unlock settlement and gross aggregation
- Time remaining based on server time
Suggested endpoints:
- GET /creator/flashdrops -> list drops with metrics
- POST /creator/flashdrops -> create drop
- PATCH /creator/flashdrops/:id -> update status, metadata
- POST /creator/flashdrops/:id/end -> end now
- GET /creator/flashdrops/metrics?range=today -> kpis

F) EDGE CASES / VALIDATION RULES (PROD GUIDANCE)
------------------------------------------------
- Price must be >= 1
- EndsIn must be >= 1 minute
- Inventory must be >= 1 for limited drops, optional for unlimited
- Status changes require permission checks
- Ending a drop prevents further unlocks
- Gross should be computed from ledger, not derived from unlock count alone
- Rate limiting & fraud checks on bursts or unusually high traffic
